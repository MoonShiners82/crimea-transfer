1. ПЛАТЁЖНЫЕ СИСТЕМЫ
================================================================================

Приоритет: ВЫСОКИЙ

1.1 YooKassa (ЮKassa)
    - Самая популярная в РФ для онлайн-платежей
    - Поддержка СБП, карт,��联, Apple Pay, Google Pay
    - API простое, хорошая документация
    - Комиссия ~2-3%
    - Реализация:
      * Создание платежа при подтверждении заявки
      * Редирект на страницу оплаты
      * Webhook для обработки статуса платежа
      * Автоматический возврат при отмене
    - Файлы: /api/payments/yookassa/route.ts

1.2 ТБанк (ex-Tinkoff)
    - Популярный эквайринг
    - Поддержка рассрочки
    - Apple Pay, Google Pay, СБП
    - Комиссия ~1.5-2.5%
    - Альтернатива YooKassa

1.3 Сбербанк Онлайн
    - Для тех, кто платит через Сбер
    - Интеграция через Сбербанк API

1.4 СБП (Система Быстрых Платежей)
    - Бесплатные переводы для физлиц
    - Можно подключить через YooKassa или напрямую

1.5 Авансовая оплата / Предоплата
    - Частичная предоплата (30-50%) при бронировании
    - Остаток — водителю наличными или картой
    - Или полная оплата онлайн со скидкой 3-5%

1.6 Подарочные сертификаты
    - Покупка сертификата на трансфер
    - Промокоды и скидки

Реализация:
  - Добавить поле paymentStatus в модель Booking (enum: unpaid, partial, paid, refunded)
  - Создать модель Payment для хранения транзакций
  - Добавить страницу /booking/payment?id=xxx
  - Обработка webhook от платёжной системы
  - Отправка чека по email/SMS

2. УВЕДОМЛЕНИЯ (NOTIFICATIONS)
================================================================================

Приоритет: ВЫСОКИЙ

2.1 SMS-уведомления
    - Подтверждение бронирования
    - Назначение водителя (ФИО, телефон, авто)
    - Напоминание за 24 часа до поездки
    - Изменение статуса
    - Провайдеры: SMS.ru, MosText, YooKassa SMS

2.2 Email-уведомления
    - Письмо с деталями бронирования
    - Чек/Invoice
    - Подтверждение оплаты
    - Письмо перед поездкой (инструкция)
    - Рассылка акций (с согласия)

2.3 Push-уведомления (PWA)
    - Service Worker + Push API
    - Мгновенные уведомления о статусе
    - Напоминания

2.4 Telegram бот
    - Уведомления для диспетчера о новых заявках
    - Уведомления для водителя о назначении
    - Уведомления для клиента о статусе
    - BotFather + webhook

Реализация:
  - Создать /lib/notifications.ts с абстракцией провайдеров
  - Добавить поле notificationPreferences в User
  - Очередь уведомлений через Vercel Cron Jobs или BullMQ

3. ОПТИМИЗАЦИЯ
================================================================================

Приоритет: СРЕДНИЙ

3.1 Производительность
    - React Server Components (RSC) где возможно
    - Lazy loading компонентов (dynamic import)
    - Оптимизация изображений (next/image + blur placeholder)
    - SWR для кэширования API-запросов на клиенте
    - Debounce для поиска в админке

3.2 База данных
    - Индексы на часто запросимые поля (уже частично сделано)
    - Connection pooling (Supabase pooler уже используется)
    - Кэширование часто запрашиваемых данных (routes, settings)
    - Очистка старых FlashCallVerification записей (cron job)

3.3 Фронтенд
    - Code splitting по роутам
    - Prefetching страниц
    - CSS-классы через Tailwind (Tree Shaking)
    - Минификация JS/CSS (Next.js делает автоматически)

3.4 SEO
    - Динамические meta-теги для каждой страницы
    - Structured data (уже есть частично)
    - Sitemap.xml (генерация через Next.js)
    - Robots.txt
    - Open Graph изображения для соцсетей

3.5 PWA
    - Service Worker для офлайн-работы
    - Manifest.json для установки на домашний экран
    - Кэширование статических ресурсов

Реализация:
  - Добавить SWR для GET-запросов
  - lazy() для тяжёлых компонентов
  - next-sitemap для генерации sitemap.xml

4.7 Админка (расширенная)

    - Управление тарифами (время суток, расстояние)
    - Управление промокодами

4.8 Рассылки
    - Email-рассылки по базе клиентов

    - Сегментация аудитории
    - A/B тестирование

4.9 Бронирование для юрлиц
    - Оформление по счету/договору
    - Авансовый отчёт
